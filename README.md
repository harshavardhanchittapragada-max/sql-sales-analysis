
SQL Sales Data Analysis
Overview

This project analyzes retail sales data using SQL to identify key business insights such as top-performing regions, products, and sales trends. The analysis is performed using SQL queries executed in SQL Server Management Studio.

The goal of this project is to demonstrate SQL skills commonly used by data analysts, including aggregation, grouping, sorting, and data exploration.

Tools Used

SQL

Microsoft SQL Server

SQL Server Management Studio (SSMS)

Dataset

The dataset contains sales transaction records including:

Product name

Sales region

Sales amount

This data is used to analyze revenue distribution and product performance.

Analysis Performed
1. View Dataset

Displays all records in the sales table.

SELECT * 
FROM sales_data;
2. Total Sales by Region

Identifies which region generates the highest revenue.

SELECT region,
SUM(sales_amount) AS total_sales
FROM sales_data
GROUP BY region
ORDER BY total_sales DESC;
3. Total Sales by Product

Shows the total revenue generated by each product.

SELECT product,
SUM(sales_amount) AS total_sales
FROM sales_data
GROUP BY product
ORDER BY total_sales DESC;
4. Total Number of Orders

Counts the total number of sales transactions.

SELECT COUNT(*) AS total_orders
FROM sales_data;
5. Orders by Region

Shows how many orders came from each region.

SELECT region,
COUNT(*) AS total_orders
FROM sales_data
GROUP BY region;
6. Highest Sales Transaction

Finds the single highest-value sales transaction.

SELECT TOP 1 *
FROM sales_data
ORDER BY sales_amount DESC;
7. Average Sales per Product

Calculates the average sales value for each product.

SELECT product,
AVG(sales_amount) AS average_sales
FROM sales_data
GROUP BY product;
Key Insights

The South region generated the highest total sales.

Laptop products recorded the highest average sales value.

Sales distribution varies across regions and products.

SQL aggregation functions help identify patterns and performance metrics quickly.
